# Story 5.3: Frontend - Real-time Progress Streaming

> **Story Type**: Feature
> **Epic**: [Deep Agent UX Integration](../epics/deep-agent-ux-integration.md)
> **Created**: 2026-01-31
> **Status**: Complete
> **Priority**: High
> **Estimated Effort**: Medium (2-3 days)
> **Depends On**: Story 5.1 (Backend Event Streaming), Story 5.2 (Accordion Tree Components)

---

## User Story

**As a** user in Deep Analysis mode,
**I want** to see real-time progress updates as the analysis runs,
**So that** I know exactly what's happening during the 3+ minute multi-agent analysis.

---

## Acceptance Criteria

### AC-1: SSE → Accordion Wiring
- [ ] New `onDeepEvent` callback added to `sendMessageStreamPersistent()` in `api.ts`
- [ ] Stream parser routes all 9 `deep_*` event types to `onDeepEvent` callback
- [ ] Events dispatched to accordion reducer via `mapDeepEventToAction()` mapper function

### AC-2: useAnalysis Hook Integration
- [ ] `useAnalysis()` accepts optional `onDeepEvent` callback parameter
- [ ] When `agent_version === "v4-deep"`, passes `onDeepEvent` to stream call
- [ ] Agent mode type widened to include `"v4-deep"` in useAnalysis signature

### AC-3: EnhancedChatInterface Wiring
- [ ] `useDeepAccordionState()` hook initialized in EnhancedChatInterface
- [ ] Deep accordion dispatch connected to useAnalysis via `onDeepEvent` callback
- [ ] `DeepAgentAccordion` component rendered when deep agent is active
- [ ] Accordion appears above the assistant message placeholder during streaming

### AC-4: Event-to-Action Mapping
- [ ] `mapDeepEventToAction()` pure function maps all 9 SSE event types to reducer actions
- [ ] Mapping handles snake_case → camelCase field name conversion
- [ ] Unit tests for all 9 event type mappings

### AC-5: Graceful Error Handling
- [ ] Unknown/malformed deep events logged but don't crash
- [ ] If accordion dispatch fails, streaming continues (errors isolated)
- [ ] Accordion state resets on new DEEP_START (idempotent restart)

### AC-6: Type Safety
- [ ] `DeepStreamEvent` type from `types/api.ts` used for all deep event handling
- [ ] No `any` types in the event mapping or dispatch chain
- [ ] TypeScript compiles cleanly with strict mode

## Technical Design

### Architecture Overview

```
Backend SSE → api.ts parser → onDeepEvent callback → mapDeepEventToAction() → dispatch()
                                                                                   ↓
                                                                          useReducer state
                                                                                   ↓
                                                                     DeepAgentAccordion UI
```

### Task 1: Add `onDeepEvent` callback to `api.ts`

**File**: `frontend/src/services/api.ts`

Add new optional parameter to `sendMessageStreamPersistent()`:

```typescript
onDeepEvent?: (event: DeepStreamEvent) => void,
```

Add routing in the stream parser (after tool_error handling):

```typescript
} else if (data.type?.startsWith("deep_") && onDeepEvent) {
  onDeepEvent(data as DeepStreamEvent);
}
```

### Task 2: Create `mapDeepEventToAction()` mapper

**New file**: `frontend/src/components/chat/deep/mapDeepEvent.ts` (~60 lines)

Pure function that maps backend SSE `DeepStreamEvent` → frontend `DeepAccordionAction`:

| SSE Event Type | Reducer Action |
|---|---|
| `deep_start` | `DEEP_START` |
| `deep_subagent_start` | `SUBAGENT_START` |
| `deep_tool_start` | `TOOL_START` |
| `deep_tool_end` | `TOOL_END` |
| `deep_subagent_result` | `SUBAGENT_RESULT` |
| `deep_debate_start` | `DEBATE_START` |
| `deep_debate_round` | `DEBATE_ROUND` |
| `deep_synthesis_start` | `SYNTHESIS_START` |
| `deep_verdict` | `VERDICT` |

Key field mappings (snake_case → camelCase):
- `subagent_name` → `subagentName`
- `tool_name` → `toolName`
- `display_name` → `displayName`
- `tool_names` → `toolNames`
- `subagent_names` → `subagentNames`
- `enable_debate` → `enableDebate`
- `has_concerns` → `hasConcerns`
- `duration_ms` → `durationMs`
- `output_preview` → `outputPreview`
- `result_summary` → `resultSummary`
- `tool_count` → `toolCount`
- `verdict_text` → `verdictText`
- `risk_level` → `riskLevel`
- `total_duration_ms` → `totalDurationMs`
- `max_rounds` → `maxRounds`
