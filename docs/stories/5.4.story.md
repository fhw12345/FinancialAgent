## User Story

**As a** user viewing a completed deep analysis,
**I want** to see a summary card with key metrics above the verdict message,
**So that** I can quickly assess the analysis quality before reading the full report.

---

## Context

The deep agent already:
- Emits `deep_verdict` event with `tool_count`, `total_duration_ms`, `risk_level` (Story 5.1)
- Renders accordion tree with live progress (Stories 5.2-5.3)
- Streams the verdict as chunk events that form the chat message bubble (pre-existing)

What's missing: A **compact summary card** that appears below the accordion (when completed) showing key metrics at a glance.

---

## Acceptance Criteria

### AC-1: Summary Card Component
- [ ] `VerdictSummaryCard` component renders below the accordion when verdict is available
- [ ] Shows: tools used count, total duration, risk level badge
- [ ] Risk level badge color-coded: HIGH (red), MODERATE (amber), LOW (green)
- [ ] Compact design: single row of metrics, not a large card

### AC-2: Integration with Accordion
- [ ] Summary card rendered inside `DeepAgentAccordion` (already has verdict data)
- [ ] Only visible when `state.verdict` is set (analysis completed)
- [ ] Smooth appearance when verdict arrives (already handled by accordion expand)

### AC-3: i18n Support
- [ ] All labels use existing `deepAgent.*` i18n keys (already defined)
- [ ] No new i18n keys needed (reuse toolsUsed, riskLevel from Story 5.2)

## Technical Design

### Observation

Looking at `DeepAgentAccordion.tsx` lines 137-163, the verdict summary section **already exists**:
```tsx
{state.verdict && (
  <div className="px-3 py-2 border-t border-gray-200 dark:border-gray-700">
    <div className="flex items-center gap-2 text-xs text-gray-500">
      <span>{t('chat:deepAgent.toolsUsed', { count: state.verdict.toolCount })}</span>
      <span>•</span>
      <span>{formatTotalDuration(state.verdict.totalDurationMs)}</span>
      {state.verdict.riskLevel && (
        <>
          <span>•</span>
          <span className={riskColor}>{t('chat:deepAgent.riskLevel', { level: ... })}</span>
        </>
      )}
    </div>
  </div>
)}
```

This story's primary work is to **extract and enhance** this existing verdict section into a slightly more prominent summary card component, and ensure the expand/collapse all toggle works.

### Task 1: Extract VerdictSummaryCard Component

**New file**: `frontend/src/components/chat/deep/VerdictSummaryCard.tsx` (~50 lines)

Extract the existing verdict section from `DeepAgentAccordion.tsx` into a dedicated component with enhanced styling:
- Slightly more prominent border and padding
- Risk level as a colored badge pill
- Tool count with wrench icon
- Duration with clock icon

### Task 2: Add Expand All / Collapse All Toggle

**File**: `frontend/src/components/chat/deep/DeepAgentAccordion.tsx`

Add a small toggle button in the verdict summary area that dispatches expand/collapse all subagents. This requires a new `EXPAND_ALL` / `COLLAPSE_ALL` action.

### Task 3: Add EXPAND_ALL / COLLAPSE_ALL Actions

**File**: `frontend/src/components/chat/deep/types.ts`

Add to `DeepAccordionAction`:
```typescript
| { type: 'EXPAND_ALL' }
| { type: 'COLLAPSE_ALL' }
```

**File**: `frontend/src/components/chat/deep/useDeepAccordionState.ts`

Add reducer cases that set all subagent expanded states to true/false.

### Task 4: Unit Tests

- Test EXPAND_ALL / COLLAPSE_ALL reducer actions
- Test VerdictSummaryCard rendering (optional, component is simple)
