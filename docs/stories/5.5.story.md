## Acceptance Criteria

### AC-1: Remove Dead Code
- [ ] Remove unused `toggleMain` and `toggleSubagent` exports from `useDeepAccordionState.ts`
- [ ] These convenience methods are never imported — `DeepAgentAccordion` creates its own
- [ ] Hook returns only `{ state, dispatch }`

### AC-2: Extract Shared Status Mappings (DRY)
- [ ] Status icon/color maps duplicated between `ToolItem.tsx` and `SubAgentSection.tsx`
- [ ] Extract into shared constants in `utils.ts`
- [ ] Both components import from shared location

### AC-3: Responsive Mobile Padding
- [ ] Accordion header: `px-4` → `px-3 sm:px-4` for tighter mobile layout
- [ ] Sub-agent sections: ensure text truncation works on narrow screens
- [ ] VerdictSummaryCard: metrics wrap gracefully on small screens

### AC-4: Unit Tests for EXPAND_ALL / COLLAPSE_ALL Interaction
- [ ] Verify toggle button dispatches correct action based on expanded state
- [ ] Tests already exist in reducer — this is about the VerdictSummaryCard render path (optional)
