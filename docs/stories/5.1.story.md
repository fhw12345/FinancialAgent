## Story

**As a** frontend developer consuming the deep agent's SSE stream,
**I want** the backend to emit structured lifecycle events in real-time as each sub-agent and tool executes,
**so that** I can build a progressive, hierarchical UI that shows users what's happening during the 3+ minute deep analysis.

---

## Acceptance Criteria

1. Deep agent emits structured SSE events for each phase transition: `deep_start`, `deep_subagent_start`, `deep_tool_start`, `deep_tool_end`, `deep_subagent_result`, `deep_debate_start`, `deep_debate_round`, `deep_synthesis_start`, `deep_verdict`
2. Each event includes relevant metadata: sub-agent name, tool name, inputs (symbol), duration_ms, status, result summary (truncated to 200 chars)
3. Streaming handler (`deep_agent.py`) forwards events to the frontend as new SSE event types (all prefixed `deep_*`)
4. Existing batch `ainvoke()` path preserved as fallback via feature flag `deep_streaming_v2` (non-breaking change)
5. Event schema documented as TypeScript type definitions (for frontend Story 5.2 to consume)
6. Backend unit tests for event emission (mocked sub-agents, assert event sequence and schema)
7. Events are sequenced with `seq` field (monotonically increasing integer) for frontend ordering

## Dev Notes

### Event Schema (TypedDict)

All events share a common base with `type`, `seq`, and `timestamp`:

```python
# backend/src/api/schemas/deep_agent_events.py
from typing import TypedDict, Literal

class DeepEventBase(TypedDict):
    type: str
    seq: int
    timestamp: str  # ISO 8601

class DeepStartEvent(DeepEventBase):
    type: Literal["deep_start"]
    symbol: str
    subagent_names: list[str]  # ["technical_analyst", "news_analyst", "financial_analyst"]
    enable_debate: bool

class DeepSubagentStartEvent(DeepEventBase):
    type: Literal["deep_subagent_start"]
    subagent_name: str
    display_name: str  # "Technical Analyst"
    icon: str  # "📊"
    tool_names: list[str]

class DeepToolStartEvent(DeepEventBase):
    type: Literal["deep_tool_start"]
    subagent_name: str
    tool_name: str
    display_name: str  # "Historical Prices"
    inputs: dict  # {"symbol": "TSLA"}

class DeepToolEndEvent(DeepEventBase):
    type: Literal["deep_tool_end"]
    subagent_name: str
    tool_name: str
    status: Literal["success", "error"]
    duration_ms: int
    output_preview: str  # first 200 chars

class DeepSubagentResultEvent(DeepEventBase):
    type: Literal["deep_subagent_result"]
    subagent_name: str
    status: Literal["success", "error"]
    duration_ms: int
    result_summary: str  # first 200 chars
    tool_count: int

class DeepDebateStartEvent(DeepEventBase):
    type: Literal["deep_debate_start"]
    round: int
    max_rounds: int

class DeepDebateRoundEvent(DeepEventBase):
    type: Literal["deep_debate_round"]
    round: int
    has_concerns: bool
    summary: str  # first 200 chars

class DeepSynthesisStartEvent(DeepEventBase):
    type: Literal["deep_synthesis_start"]

class DeepVerdictEvent(DeepEventBase):
    type: Literal["deep_verdict"]
    verdict_text: str  # full verdict section
    risk_level: str | None  # "HIGH", "MODERATE", etc. if extractable
    tool_count: int
    total_duration_ms: int
```

[Source: New schema, extends pattern from `backend/src/api/schemas/chat_models.py`]

### Callback + Queue Architecture

The callback runs synchronously inside `analyze()` (which is async but sub-agent invocation blocks). Use `asyncio.Queue` to bridge:

```python
# In streaming handler
event_queue: asyncio.Queue[dict] = asyncio.Queue()

def on_event(event: dict) -> None:
    event_queue.put_nowait(event)

# Pass to adapter
result_task = asyncio.create_task(
    agent.ainvoke(user_message, on_event=on_event, ...)
)

# SSE generator reads from queue
while not result_task.done() or not event_queue.empty():
    try:
        event = await asyncio.wait_for(event_queue.get(), timeout=1.0)
        yield format_sse_event(event)
    except asyncio.TimeoutError:
        continue  # Keep polling

# After result_task completes, stream verdict chunks
result = await result_task
# ... stream verdict text as chunks
```

[Source: Pattern adapted from `backend/src/api/chat/streaming/react_agent.py` tool event streaming]

### Tool Interception Strategy

To capture tool_start/tool_end events from inside sub-agents, two approaches:

**Option A (Recommended): LangGraph callback handler**
- Create a custom `BaseCallbackHandler` subclass that captures `on_tool_start` and `on_tool_end`
- Pass to `_invoke_subagent()` via `config["callbacks"]`
- Pro: Clean, no modification to sub-agent code. Con: May not capture all tool metadata.

**Option B: Wrapper around tool functions**
- Wrap each tool in an event-emitting decorator before passing to sub-agent
- Pro: Full control over inputs/outputs. Con: More invasive.
