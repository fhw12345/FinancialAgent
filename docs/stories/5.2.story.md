## Story

**As a** user watching a deep agent analysis execute,
**I want** to see a collapsible accordion tree showing sub-agents and their tools organized hierarchically,
**so that** I can understand what the deep agent is doing at each stage and drill into details on demand.

---

## Acceptance Criteria

### AC-1: DeepAgentAccordion Component
- [ ] `DeepAgentAccordion` component renders a nested tree structure
- [ ] Top level: main analysis card with symbol, status, total duration
- [ ] Second level: sub-agent cards (Technical Analyst, News Analyst, Financial Analyst)
- [ ] Third level: individual tool execution items under each sub-agent
- [ ] Fourth level (optional): Debate Verification section

### AC-2: Sub-Agent Card Rendering
- [ ] Each sub-agent card shows: icon (from `DeepStreamEvent`), display_name, status badge
- [ ] Status states: pending (gray), running (blue spinner), completed (green check), failed (red X)
- [ ] Collapsed view shows: icon + name + status + tool count + duration
- [ ] Expanded view shows: tool list + result_summary preview (first 200 chars)

### AC-3: Tool Item Rendering
- [ ] Each tool item shows: tool display_name, status indicator, duration_ms
- [ ] Running state: animated spinner (consistent with existing `ToolExecutionProgress`)
- [ ] Completed state: checkmark + duration badge (e.g., "1.2s")
- [ ] Error state: red X + error message
- [ ] Input preview tooltip on hover (e.g., "Symbol: TSLA")

### AC-4: Expand/Collapse Behavior
- [ ] Each accordion level expands/collapses independently
- [ ] Click on header toggles that level
- [ ] Chevron icon rotates on expand (consistent with existing `ToolMessageWrapper`)
- [ ] Smooth CSS transition on expand/collapse (200ms)
- [ ] Default state: main level expanded, sub-agents expanded, tools collapsed

### AC-5: Accessibility
- [ ] Keyboard navigation: Tab to focus, Enter/Space to toggle
- [ ] ARIA attributes: `role="region"`, `aria-expanded`, `aria-controls`
- [ ] Screen reader announces expand/collapse state changes

### AC-6: i18n Support
- [ ] All labels translated in `en/chat.json` and `zh-CN/chat.json`
- [ ] New i18n namespace: `chat:deepAgent.*`
- [ ] Labels: status names, phase names, "tools used", "duration", etc.

### AC-7: Performance
- [ ] Component handles 3-5 sub-agents with 2-6 tools each without jank
- [ ] Uses `React.memo` on sub-agent and tool item components
- [ ] No unnecessary re-renders when sibling sections update

## Technical Design

### Component Hierarchy

```
DeepAgentAccordion (top-level container)
├── DeepAgentHeader (symbol, overall status, expand-all toggle)
├── SubAgentSection (one per sub-agent)
│   ├── SubAgentHeader (icon, name, status, tool count)
│   ├── ToolItem[] (one per tool, only visible when expanded)
│   └── SubAgentResult (summary text, visible when completed)
├── DebateSection (optional, when enable_debate=true)
│   ├── DebateHeader (icon, round info)
│   └── DebateRound[] (round number, concerns, summary)
└── SynthesisIndicator (shows when synthesis is in progress)
```

### State Shape

```typescript
interface DeepAccordionState {
  symbol: string;
  status: 'pending' | 'running' | 'completed' | 'failed';
  subagents: Map<string, SubAgentState>;
  debate: DebateState | null;
  synthesis: { started: boolean };
  verdict: VerdictState | null;
  expanded: {
    main: boolean;
    subagents: Record<string, boolean>;
    tools: Record<string, boolean>;
  };
}

interface SubAgentState {
  name: string;
  displayName: string;
  icon: string;
  status: 'pending' | 'running' | 'completed' | 'failed';
  toolNames: string[];
  tools: Map<string, ToolState>;
  resultSummary: string;
  durationMs: number;
  toolCount: number;
}

interface ToolState {
  name: string;
  displayName: string;
  status: 'pending' | 'running' | 'completed' | 'failed';
  durationMs: number;
  outputPreview: string;
  inputs: Record<string, unknown>;
}
```

## Dev Notes

- Follow existing patterns: `React.memo` for leaf components, CSS-only animations
- Use same icon library as `ToolExecutionProgress` (Lucide React)
- Component should accept events array as props (pure rendering, no SSE awareness)
- Reducer must handle idempotent updates (same event applied twice = no change)
- Default expanded state: main=true, subagents=true, tools=false
